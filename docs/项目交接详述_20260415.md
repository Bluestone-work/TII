# multi-robot-exploration-rl 项目交接详述

更新时间：2026-04-15  
项目根目录：`/home/wj/work/multi-robot-exploration-rl`

## 1. 项目概述

本项目是一个基于 ROS2 Humble + Gazebo Classic + Python 强化学习框架的多机器人导航与避碰实验工作区，目标是支持以下几类研究与工程任务：

1. 多机器人强化学习训练。
2. GNN / MAPPO / MLP / IPPO / SB3 PPO 等不同算法对比。
3. ORCA / DWA 等传统方法对比与数据采集。
4. TurtleBot3 多机器人仿真环境构建。
5. 训练后策略在仿真或实机上的部署验证。

当前最主要、最常用、维护最活跃的主线是：

1. `start_rl_environment_tb3`：TurtleBot3 多机器人 Gazebo 环境。
2. `gnn_marl_training`：GAT / MLP 版本的 MAPPO 训练、测试、部署。
3. 根目录 `run_curriculum.sh` / `run_parallel.sh` / `run_test.sh`：训练与测试主入口。

## 2. 技术栈与运行环境

建议默认环境：

- Ubuntu 22.04
- ROS2 Humble
- Gazebo 11
- Python 3.10
- Conda 环境名：`ros2`
- PyTorch
- Ray / RLlib
- TurtleBot3 相关 ROS 包

推荐启动方式：

```bash
source /home/wj/anaconda3/etc/profile.d/conda.sh
conda activate ros2
source /opt/ros/humble/setup.bash
cd /home/wj/work/multi-robot-exploration-rl
source install/setup.bash
export TURTLEBOT3_MODEL=burger
export GAZEBO_IP=127.0.0.1
```

说明：

- `GAZEBO_IP=127.0.0.1` 用于减少长时间训练时 Gazebo multicast 网络告警。
- 若只在本机训练，建议优先使用 headless 模式，避免 RViz / GUI 长时间占用资源。

## 3. 当前主线架构

当前主线调用链建议理解为：

1. 根目录 `run_parallel.sh`：并行启动 MLP / GAT 对比实验。
2. 根目录 `run_curriculum.sh`：薄包装，转发到 `src/gnn_marl_training/run_curriculum.sh`。
3. `src/gnn_marl_training/run_curriculum.sh`：负责课程学习调度、重启 Gazebo、启动 RLlib 训练。
4. `src/gnn_marl_training/gnn_marl_training/train_gnn_mappo_full.py`：核心训练脚本。
5. `src/gnn_marl_training/gnn_marl_training/gnn_marl_env.py`：多机器人环境封装。
6. `src/start_rl_environment_tb3`：提供地图、world、launch、动态障碍、TurtleBot3 模型。

如果接手人只想快速上手，应先掌握这一条链路。

## 5. `src/` 下各功能包说明

### 5.1 `src/start_rl_environment_tb3`

当前仿真主环境包，所有 GNN-MAPPO / MLP / 测试主线都优先使用它。

主要作用：

- 启动 Gazebo 世界。
- 启动多台 TurtleBot3 Burger。
- 加载地图与世界文件。
- 启动动态障碍控制节点 `obstacle_mover.py`。
- 提供 RViz 配置与 headless launch。

关键子目录：

| 子目录/文件 | 作用 |
|---|---|
| `launch/main.launch.py` | 有 GUI / RViz 的主 launch |
| `launch/main_headless.launch.py` | 无 GUI 训练版主 launch |
| `launch/start_world.launch.py` | 只启动世界 |
| `launch/start_robots.launch.py` | 只启动机器人 |
| `worlds/` | Gazebo 世界文件 |
| `maps/` | 地图 png/pgm/yaml 以及旧版环境辅助文件 |

备注：

- 这是当前训练主线最重要的环境包。

- 动态障碍默认速度基准已调整为 `0.11 m/s`，约为机器人线速度上限的一半。

当前强化学习主线包。

主要作用：

- 提供 GAT / MLP 版本的 MAPPO 训练。
- 提供课程学习训练脚本。
- 提供 checkpoint 测试脚本。
- 提供训练后部署节点 `robot_policy_node`。

关键文件：

| 文件 | 作用 |

| `gnn_marl_training/train_gnn_mappo_full.py` | 主训练脚本 |
| `gnn_marl_training/test_gnn_mappo.py` | checkpoint 推理测试 |
| `gnn_marl_training/gnn_marl_env.py` | 多智能体环境封装 |

## 10. 部署使用说明

### 10.1 仿真内在线部署

`gnn_marl_training` 提供 `robot_policy_node`，可以让单台机器人独立加载训练好的策略并通过 ROS2 topic 控制。

示例：

```bash
ros2 run gnn_marl_training robot_policy_node \
  --ros-args \
  -p robot_id:=0 \
  -p num_agents:=4 \
  -p checkpoint_path:=/path/to/checkpoint_or_policy.pt \
  -p communication_range:=3.5 \
  -p max_neighbors:=3 \
  -p goal_x:=5.0 \
  -p goal_y:=5.0
```

作用：

- 订阅 `/tb3_i/scan` 和 `/tb3_i/odom`
- 广播 `/gnn_swarm/robot_i/state`
- 推理后发布 `/tb3_i/cmd_vel`

### 10.2 实机部署思路

如果后续用于实机 TurtleBot3，多机部署最基本流程是：

1. 每台机器人安装 ROS2 与项目依赖。
2. 每台机器人运行本机的 `robot_policy_node`。
3. 保证所有机器人能通过 ROS2 DDS 互相通信。
4. 每台机器人设置不同的 `robot_id`，相同的 `num_agents`。
5. 统一模型权重、通信半径、观测配置。

交接注意：

- 当前仓库更偏“仿真训练 + 仿真测试”，实机部署节点已有，但整套实机 launch 不是当前文档的主线。
- 若要上实机，需单独补充网络配置、DDS 配置、时钟同步和安全停机方案。

## 14. 常见问题与排障建议

### 14.1 Gazebo 长训后出现 multicast 网络告警

现象：

```text
Exception sending a multicast message: Network is unreachable
```

建议：

```bash
export GAZEBO_IP=127.0.0.1
```

并尽量使用：

```bash
--render_mode headless
```

### 14.2 GPU 没有被使用

原因：

- `run_parallel.sh` 默认 `RAY_NUM_GPUS_PER_JOB=0`。

处理：

```bash
./run_parallel.sh ... --ray_num_gpus_per_job 1 --cuda_visible_devices 0
```

### 14.3 动态障碍不移动

当前主线已修复 `main_headless.launch.py`，如果再次出现该问题，优先检查：

1. 是否真的走了 `start_rl_environment_tb3`。
2. 是否是 headless launch。
3. `obstacle_mover.py` 是否启动。

### 14.4 失败不重置

当前主线已在 `gnn_marl_env.py` 中把 `truncated` 纳入 terminal 判定。若再次出现，优先查：

1. `collision` 是否被传感器真正检测到。
2. `stage*_ros.log` 是否出现 bumper/contacts 异常。

## 15. 建议的接手顺序

建议新接手同学按以下顺序理解项目：

1. 看 `src/start_rl_environment_tb3`，确保知道环境如何启动。
2. 看 `run_parallel.sh`，理解主线训练入口。
3. 看 `src/gnn_marl_training/run_curriculum.sh`，理解训练流程和课程阶段。
4. 看 `train_gnn_mappo_full.py`，理解参数和训练配置。
5. 看 `gnn_marl_env.py`，理解观测、奖励、reset、通信图。
6. 再看 `mappo_mlp_model.py` 和 `gat_rllib_model.py`，理解模型结构。
7. 最后再看 `run_test.sh` 和 `robot_policy_node.py`，理解测试与部署。

## 16. 交接结论

这个仓库不是一个“单一算法项目”，而是一个多分支、多阶段演化出来的多机器人导航实验工作区。  
当前真正应当视为主线的，是 TurtleBot3 环境包加 `gnn_marl_training` 主链路。其余许多目录和脚本仍有参考价值，但不应在交接初期与主线混为一谈。

如果只保留一句话给接手人：

> 先用 `run_parallel.sh / run_curriculum.sh / run_test.sh` 跑通 `start_rl_environment_tb3 + gnn_marl_training`，确认主线稳定后，再回头整理 SB3、ORCA、BC 和其他实验分支。
