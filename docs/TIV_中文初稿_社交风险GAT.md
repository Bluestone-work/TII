# 面向动态多机器人导航的社交风险图增强 MAPPO：一种无需显式通信的关系建模框架

注：本文档为中文初稿，目标是按照 IEEE Transactions on Intelligent Vehicles（TIV）常见论文写法组织内容。当前版本优先完成论文结构、方法叙述与实验设计，定量结果、图表编号、参考文献格式和英文润色可在后续版本中补齐。

## 摘要

多机器人在狭窄通道、交汇路口和动态障碍密集场景中的安全导航，面临局部感知受限、交互关系强耦合以及策略泛化困难等问题。传统 ORCA 等解析式方法具备较好的可解释性和避碰几何先验，但在复杂动态场景中难以同时兼顾全局任务效率、场景适应性与可学习协同行为。基于强化学习的多机器人导航方法具有更强的策略表达能力，但直接将图神经网络用于策略主干时，容易把局部场景理解、邻居交互建模和通信噪声鲁棒性混合在一起，从而造成训练不稳定和性能退化。

针对上述问题，本文提出一种社交风险图增强的 MAPPO 框架。该方法保留 MAPPO-MLP-LSTM 的局部导航主干，使机器人对静态障碍、目标方向和短时动态风险的理解路径与稳定基线保持一致；同时额外构建一条轻量级 social-risk GAT 分支，仅基于真值式社交风险 token 建模机器人之间的交互关系，并通过后置残差门控将关系特征注入策略表示，而不接管主干导航功能。为了提升训练稳定性，本文进一步采用集中训练-分散执行范式、阶段式课程学习和面向 GAT 的保守 PPO 超参数配置。

建议在四阶段动态难度环境中，将所提方法与 MAPPO-MLP-LSTM、基于邻居通信图的旧版 GAT 以及 ORCA 基线进行系统比较，并从成功率、碰撞率、平均完成时间、死锁率、最小社交间距和推理时延等多个维度进行评估。预期结果是，所提方法能够在不依赖显式通信的条件下，显著降低碰撞与死锁，并在动态障碍和多机器人会车场景中取得优于 MLP 基线和解析式基线的综合性能。

关键词：多机器人导航，MAPPO，图注意力网络，社交风险建模，课程学习，动态障碍避碰

## 1. 引言

随着仓储物流、园区配送和服务机器人系统的发展，多机器人在共享空间中的安全协同导航已成为智能车辆与移动机器人领域的重要研究课题。与单机器人导航不同，多机器人系统不仅需要处理静态障碍和目标到达问题，还必须在局部感知条件下应对会车、汇入、交叉口争抢和动态障碍干扰等强交互情形。当机器人数量增加、环境拓扑变得狭窄或动态障碍密度上升时，局部避碰策略的稳定性和可扩展性将直接决定系统能否落地。

一类代表性方法是 ORCA 等解析式多智能体避碰算法。这类方法以相对位置、相对速度和时间裕量为核心，具有计算高效、无需训练、可解释性强等优点。然而，ORCA 主要解决局部速度选择问题，难以直接利用复杂场景语义、历史观测和任务上下文，也较难在动态障碍密集、局部最优频发或长期任务目标约束较强的情形下获得最优综合行为。

另一类方法是基于深度强化学习的多机器人导航。其优势在于可以通过数据驱动方式学习复杂的场景适应策略，并通过集中训练-分散执行提升多智能体协同能力。尤其是 MAPPO 一类方法，兼顾了训练稳定性和多智能体场景下的实用性。然而，标准 MLP 策略往往难以显式刻画机器人之间的关系结构，而直接引入 GNN/GAT 虽然能够增强交互建模能力，却容易带来新的不稳定因素：如果图分支同时承担局部场景建模、邻居关系聚合和通信噪声鲁棒性，训练目标会明显耦合，导致策略长期处于高熵、低收益状态。

本文工作的出发点是：在多机器人动态导航任务中，图网络应优先承担“关系建模”而非“全部建模”。换言之，局部导航主干应尽可能保持稳定、可复用和与基线公平一致，而 GAT 分支只负责回答“周围机器人对当前决策形成了怎样的社交风险结构”这一更聚焦的问题。基于这一思路，本文将图分支改造为 social-risk GAT：不再直接拟合噪声通信链路，而是基于环境中可计算的真值式社交风险 token 构造 ego-centric social graph，用轻量关系建模增强策略，而不是替代策略主干。

本文的主要贡献如下：

1. 提出一种社交风险图增强 MAPPO 框架，将图注意力网络从“策略主干”退回到“关系增强分支”，显著降低训练耦合度。
2. 保留与 MAPPO-MLP-LSTM 一致的局部导航主干，仅使用 GAT 建模机器人之间的社交风险关系，从而保证方法比较的公平性与收敛稳定性。
3. 设计基于真值式 social-risk token 的 ego-centric 图构建方式，使模型在不依赖显式通信的前提下仍能学习 agent-agent 风险关系。
4. 给出一套适合 TIV 风格论文的实验设计方案，包括主结果实验、消融实验、泛化实验、鲁棒性实验和效率评估，为后续完整投稿提供可直接执行的评测框架。

## 2. 相关工作

### 2.1 解析式多机器人避碰

ORCA、RVO 等方法通过构造速度障碍和线性约束，在共享责任假设下为每个机器人求解可行速度，代表了分布式多机器人避碰的经典路线。这类方法具备无需训练、理论几何意义清晰和推理开销低等优点，是本文的重要参考基线。但解析式方法通常依赖局部几何规则，在复杂动态干扰、长期任务约束和多模态行为决策上存在局限。

### 2.2 基于强化学习的多机器人导航

近年来，深度强化学习被广泛用于单机器人和多机器人导航。与解析式方法相比，强化学习可联合利用目标、激光、局部风险和历史信息，从而学习更复杂的行为模式。MAPPO 由于其训练稳定、实现成熟和集中训练-分散执行的优势，已成为多智能体强化学习中的强基线。然而，标准 MAPPO 通常依赖 MLP/RNN 处理局部观测，对邻居关系的结构化表示能力有限。

### 2.3 图神经网络与多智能体关系建模

GNN/GAT 为多智能体交互提供了一种天然的关系建模工具。已有研究表明，图结构能够帮助策略识别重要邻居并学习局部协同。然而，图模型在实际系统中常与通信图、动态图、噪声信道和复杂主干耦合，导致训练路径过长、超参数敏感和复现困难。本文与这类工作不同之处在于：我们不让 GAT 主导局部导航，而是让其只聚焦于社交风险关系建模，并使用与 MLP 相同的局部主干保证公平比较。

## 3. 问题定义

本文考虑一个多机器人动态导航任务。环境中存在 $N$ 个差分驱动移动机器人、静态障碍和可移动动态障碍。每个机器人在局部可观测条件下，根据激光雷达、目标相对位姿、自身速度、短时风险特征以及社交风险 token 输出动作。系统目标是在保证低碰撞率和低死锁率的前提下，提高任务完成率并缩短到达时间。

在集中训练-分散执行设置下，第 $i$ 个机器人在时刻 $t$ 的局部观测记为 $o_i^t$，全局状态记为 $s^t$。策略 $\pi_\theta(a_i^t|o_i^t)$ 仅依赖本地观测执行，而 critic 使用全局状态估计价值函数 $V_\phi(s^t)$。本文关注的关键问题是：如何在不破坏局部导航主干稳定性的前提下，将 robot-robot 交互关系显式注入策略，以提高动态场景下的避碰与协同能力。

## 4. 方法

### 4.1 总体框架

本文采用 MAPPO 作为训练框架，并构建两个策略分支：

1. 局部导航主干：结构与 MAPPO-MLP-LSTM 一致，输入为本地观测与邻居槽位，负责学习目标驱动、障碍规避和时序记忆。
2. 社交风险 GAT 分支：仅使用真值式 social-risk token 构图，学习机器人之间的交互风险关系。

二者在策略末端通过残差门控融合。这样，局部导航能力不会依赖图分支收敛，图分支只在确有必要时对策略产生修正。

### 4.2 局部观测与基础 MAPPO 主干

当前项目中的局部观测主要由以下部分组成：

1. 多帧激光扇区历史，用于表示机器人周围局部几何结构。
2. 目标在机器人坐标系中的相对位置。
3. 机器人自身线速度与角速度。
4. 前方风险、左右风险、近障惩罚等安全特征。
5. 面向短时动态障碍的 predictive 特征。
6. neighbor prediction 特征，即真值式社交风险 token。
7. 障碍物运动特征。

在 MLP 基线中，actor 主干为 `LayerNorm + LSTMCell + Linear head`。本文保留这一结构，使新的 GAT 方法和 MLP 基线在局部导航路径上尽可能一致。这样做的原因是：若主干结构同时改变，则实验结果无法区分性能收益究竟来自图建模还是来自更换了主干网络。

### 4.3 社交风险 token 与 social graph 构造

本文不再优先依赖邻居通信图，而是使用环境中直接计算得到的 social-risk token 构图。每个 token 对应一个潜在交互机器人，包含如下语义信息：

1. 归一化后的相对位置。
2. 接近速度或闭合速度 proxy。
3. 归一化 TTC 指标。
4. 最小间距 proxy。
5. 聚合后的风险分数。

对于每个机器人，我们选择风险最高的 top-$K$ 个 social-risk token，与 ego 节点共同构成一个 ego-centric social graph。图中仅保留 ego 与各风险节点之间的双向边和自环。边权不通过显式通信质量建模，而通过风险、接近速度和 TTC 等量构造 attention bias，从而引导 GAT 关注最危险的交互关系。

这种设计与 ORCA 思路存在联系：二者都强调相对位置、相对速度和碰撞时间等社交几何变量的重要性；区别在于，ORCA 通过解析约束直接求速度，本文则通过学习式关系编码来调制策略表示。

### 4.4 Social-risk GAT 分支

在 social-risk GAT 分支中，ego 节点首先由局部观测编码得到上下文表示，risk token 通过共享 MLP 编码为节点特征。随后使用两层 GAT 对 graph 进行聚合，得到 ego 节点的社交关系嵌入。该嵌入不直接生成动作，而是通过一个投影层映射到与 LSTM 输出同维度的 social feature。

这里的设计原则是“图分支只做关系增强，不做决策接管”。因此 social feature 只在策略末端参与融合，避免图分支训练不稳定时破坏基础导航能力。

### 4.5 后置残差门控融合

设 MLP-LSTM 主干输出为 $h_i$，social-risk GAT 输出为 $g_i$。本文使用如下方式进行融合：

$$
\tilde{h}_i = \text{LN}(h_i + \sigma([h_i, g_i]) \odot \Delta(g_i)),
$$

其中 $\sigma(\cdot)$ 为门控函数，$\Delta(\cdot)$ 为轻量 MLP 投影。该结构具有两点优势：

1. 当图分支学习不足时，门控可以自动减弱其影响，维持主干稳定。
2. 当图分支识别出高风险交互时，可以通过残差方式对动作决策进行局部修正。

### 4.6 Critic 与训练设置

critic 默认采用集中式 MLP，而非 GAT critic。原因是本文关注 actor 的关系建模改进，而不是进一步增加 critic 的结构复杂度。集中训练-分散执行仍然保持 MAPPO 设定：actor 在执行时仅需本地观测，critic 在训练时访问全局状态。

### 4.7 课程学习与训练稳定策略

项目中采用四阶段课程学习环境：

1. Stage 1：`warehouse_aisles`，无动态障碍，先学会平滑前进。
2. Stage 2：`corridor_swap`，少量动态障碍，开始引入避碰压力。
3. Stage 3：同走廊交换地图，中高密度动态障碍。
4. Stage 4：高密度高速动态障碍的最终任务场景。

对于社交风险 GAT 版本，建议保持如下训练原则：

1. 默认使用 `critic_mode=mlp`。
2. 默认不启用噪声通信图训练。
3. GAT 版本使用比 MLP 更保守的 PPO 配置，如更低的 entropy coefficient 和更小的 clip 参数。
4. 所有方法使用相同的课程链路、动作空间和奖励结构，以保证比较公平。

## 5. 实验设计

本节重点回答一个问题：对于当前项目，什么实验最合理、最像 TIV 风格、同时又能直接支撑论文主张。

### 5.1 实验目标与研究问题

建议将实验问题明确写成以下四类：

1. 所提 social-risk GAT 是否优于 MLP 基线？
2. 性能提升是否确实来自社交关系建模，而不是额外参数或训练技巧？
3. 所提方法在机器人数量增加、动态障碍增强和地图切换时是否具有更强泛化性？
4. 所提方法是否仍保持可接受的训练和推理成本？

### 5.2 对比方法

最合理的对比方法不是越多越好，而是要能回答核心问题。建议至少包含以下四组：

1. `MAPPO-MLP-LSTM`：主基线。
2. `Social-Risk GAT (ours)`：本文方法。
3. `Neighbor-Communication GAT`：旧版基于邻居通信图的 GAT，用于说明为什么“直接学通信图”不如“只学社交风险关系”。
4. `ORCA`：解析式基线，用于体现传统局部避碰方法与学习式方法的差异。

如果时间有限，最少也应完成 1、2、4 三组。

### 5.3 评估场景

建议实验场景直接沿用项目现有课程环境，以保证工程与论文一致：

1. Stage 1：静态入门场景。
2. Stage 2：轻量动态场景。
3. Stage 3：中高密度动态障碍场景。
4. Stage 4：最终高难度场景。

此外，建议加入一个额外泛化场景：

1. 训练中未见的地图拓扑，或固定 Stage 配置下改变初始位姿分布。

### 5.4 评价指标

建议采用以下指标，其中前五项应作为主指标：

1. 成功率（Success Rate）。
2. 碰撞率（Collision Rate）。
3. 平均完成时间或平均 episode length。
4. 死锁率（Deadlock Rate）。
5. 最小机器人间距（Minimum Inter-robot Distance）。
6. 平均累计回报。
7. 推理时延（单步 forward 时间）。
8. 参数量与训练 wall-clock 时间。

如果篇幅允许，可增加：

1. 社交冲突场景下的局部避碰成功率。
2. 窄通道会车成功率。
3. 机器人数量扩展时的性能退化斜率。

### 5.5 最合理的主实验

这是最关键的一组实验，建议作为论文主表。

#### 实验 E1：主结果对比

目的：验证 social-risk GAT 是否在核心任务上优于 MLP 和 ORCA。

设置：

1. 方法：MLP、Social-Risk GAT、ORCA、Neighbor-Communication GAT。
2. 机器人数量：先固定为 4。
3. 测试场景：Stage 1 到 Stage 4 全部测试。
4. 每种方法至少 3 个随机种子；若资源允许，建议 5 个种子。
5. 每个 checkpoint 在每个阶段至少测试 20 至 50 个 episode。

推荐输出：

1. 一个主表：四个阶段上的成功率、碰撞率、完成时间、死锁率。
2. 一张训练曲线图：MLP vs Social-Risk GAT 的训练回报和碰撞率曲线。

如果只能做一组最重要实验，就做这一组。

#### 实验 E2：与旧版通信 GAT 的直接比较

目的：验证“图网络只学社交关系”优于“图网络直接承担邻居通信建模”。

设置：

1. 方法：Neighbor-Communication GAT vs Social-Risk GAT。
2. 其他训练配置保持一致。
3. 对比训练稳定性、熵变化、成功率和碰撞率。

推荐输出：

1. 熵-回报联合曲线。
2. Stage 3 和 Stage 4 上的主指标对比表。

这一实验对于支撑本文方法设计动机非常重要，建议作为核心消融的一部分。

### 5.6 最合理的消融实验

#### 实验 E3：结构消融

目的：证明性能来自“社交关系建模”，而不是简单增加模型容量。

建议消融项：

1. `w/o GAT`：直接退化为 MLP 主干。
2. `w/o gate`：将门控残差改为简单加法融合。
3. `w/o residual`：只保留图特征线性拼接。
4. `GAT critic`：将 critic 从 MLP 改为 GAT，验证是否值得增加复杂度。

推荐输出：

1. Stage 4 上的消融表。
2. 一张图显示不同结构的训练稳定性差异。

#### 实验 E4：风险信息源消融

目的：验证真值式 social-risk token 的价值。

建议设置：

1. 使用 social-risk token。
2. 使用 raw neighbor token。
3. 同时使用 social-risk token 与 raw neighbor token。
4. 去掉 risk bias，只保留普通注意力。

这一实验可以回答：模型究竟依赖什么信息源做出社交避碰。

### 5.7 最合理的泛化与鲁棒性实验

#### 实验 E5：机器人数量扩展

设置：

1. 在 2、4、6 台机器人上测试。
2. 训练时可固定 4 台机器人，测试时外推到 2 和 6 台。

观察点：

1. 性能是否随机器人数量增长平滑退化。
2. Social-Risk GAT 是否比 MLP 具有更好的扩展性。

#### 实验 E6：动态障碍强度鲁棒性

设置：

1. 固定地图和机器人数量。
2. 改变动态障碍数量和速度，例如低、中、高三档。

观察点：

1. 不同方法在高动态风险条件下的性能退化幅度。
2. Social-Risk GAT 是否在强干扰下仍能保持较低碰撞率。

### 5.8 最合理的效率实验

#### 实验 E7：工程成本评估

目的：符合 TIV 对工程可用性的关注。

建议报告：

1. 模型参数量。
2. 单步推理时间。
3. 单次训练 wall-clock 时间。
4. 训练收敛所需的环境步数。

这组实验不需要篇幅很大，但最好给一张表。否则审稿人容易质疑“性能提升是否只是靠更大模型换来的”。

### 5.9 建议的实验优先级

如果时间有限，最合理的优先级是：

1. E1 主结果对比。
2. E3 结构消融。
3. E5 机器人数量扩展。
4. E7 工程成本评估。
5. E2 与旧版通信 GAT 比较。
6. E6 动态障碍强度鲁棒性。
7. E4 风险信息源消融。

换句话说，最少要保证“主结果 + 一个消融 + 一个泛化 + 一个效率表”。

## 7. 讨论

本文方法与 ORCA 并非完全对立。相反，social-risk GAT 的设计本质上吸收了 ORCA 强调相对位置、相对速度和碰撞时间裕量的思想，只是将解析约束替换为学习式关系编码。因此，本文更适合将自己定位为“受 ORCA 启发的学习式社交风险建模框架”，而不是简单声称“完全替代 ORCA”。

同时，本文方法也存在局限。首先，当前 social-risk token 来自环境中的真值式计算，仍需进一步讨论从真实传感器估计这些量的误差传播问题。其次，尽管方法在训练期不依赖显式通信，但部署到真实多机器人系统时仍需考虑感知同步、定位误差和控制延迟带来的影响。最后，课程学习、奖励设计和环境构造对最终表现仍有较大影响，这要求在论文中对训练协议进行充分公开。

## 8. 结论

本文围绕动态多机器人导航任务，提出了一种社交风险图增强 MAPPO 框架。与直接将 GAT 用作策略主干不同，本文保留了稳定的 MLP-LSTM 局部导航路径，并将 GAT 约束为一个只负责 robot-robot 风险关系建模的轻量增强分支。该设计在方法上更接近 ORCA 的社交几何直觉，同时保留了强化学习在复杂场景中的表达优势。

从论文写作角度看，最合理的实验路线是：先完成 MLP、Social-Risk GAT 和 ORCA 的主结果对比，再用旧版通信 GAT 和结构消融解释方法设计动机，最后通过机器人数量扩展、动态障碍鲁棒性和效率分析支撑方法的泛化性与工程价值。按照这一组织方式，本文较容易形成一篇结构完整、问题明确、实验闭环清晰的 TIV 风格论文。
