# 8 车密集协调训练改进方案(对比参考仓库 navigation-mappo-rl)

> 背景:Stage1→6 curriculum 中,4 车(Stage2 峰值 +145)、6 车(Stage3 峰值 +141)
> 收敛良好,但 **8 车(Stage4)两次都到不了正回报**。本文档对比开源参考实现
> [avbiswas/navigation-mappo-rl](https://github.com/avbiswas/navigation-mappo-rl),
> 从任务、采样、奖励、网络、训练五个层面定位问题并给出改进方案。

---

## 一、核对结论:8 车跑不好,主因不是"场地太挤"

最初猜测"8 车把场地塞爆、物理接近无解"。**密度量化后该结论被推翻**:

| 指标 | 参考仓库 circle(能跑 8 车) | 本项目 circle_swap 8 车 |
|---|---|---|
| 车占地 / 场地面积 | 1.57% | **1.27%(更空)** |
| 相邻车间距 / 车直径 | 4.9 倍 | **5.7 倍(更宽松)** |
| 排布方式 | 半径 0.25 圆,8 车对角换位 | 半径 ~2.6m 圆,8 车对角换位 |

**本项目 8 车场地比能成功跑 8 车的参考仓库还要宽松。** 所以瓶颈不在空间,
而在下面几个被忽视的层面。

---

## 二、真因之一:起终点采样的几何冲突(11100 次"非法路线")

### 现象
Stage4 训练日志中"⚠️ 强制路线非法,回退随机安全采样"出现 **11100 次**
(4 车 Stage2 仅 3282 次),平均每 ~34 步就触发一次。

### 根因(代码定位:`gnn_marl_env.py` reset / `_is_valid_start_goal_pair`)
8 车对角换位时:
- **8 个起点 + 8 个终点 = 16 个点全部落在同一个半径 2.6m 的圆上**
- 圆周长 16.3m,16 点平均间距仅 **1.02m**,恰好卡在 `min_agent_sep=1.0` 阈值
- map8 额外叠加 `±0.3m 随机扰动`(`reset()` 中 `map_number==8` 分支)
- 结果:大量采样违反 `min_agent_sep` / `min_goal_sep=2.0` → 判非法 → 回退随机采样

### 危害
回退随机采样后,**每个 episode 的起终点都在变,任务分布不一致**。智能体面对的
不是一个固定的"8 车对角换位"任务,而是被采样器搅乱的、时好时坏的任务集合。
**这是 8 车学不稳的隐藏元凶——算法再好,任务分布不稳也学不出来。**

### 改进
1. **放宽/移除 map8 的 ±0.3m 扰动**(8 车时扰动是压垮采样的最后一根稻草),或把扰动
   降到 ±0.1m。
2. **降低 `min_agent_sep`**:8 车圆周排布时 1.0m 太苛刻,改为 0.6~0.8m,或按车数自适应
   `min_agent_sep = min(1.0, 0.7 * 圆周/(2*车数))`。
3. **改用固定的对角配对起终点**(像参考仓库那样预定义 8 个 antipodal pair),不做随机
   重采样——保证任务分布一致。这是最干净的做法,也利于实验复现。
4. 监控回退率:在 reset 里统计非法率,若 > 5% 应告警,提示采样配置有问题。

## 三、真因之二:奖励尺度过大,价值函数难拟合

### 对比
| | 参考仓库 | 本项目 |
|---|---|---|
| 到达奖励 | **+10** | (回报量级到 ±150) |
| 碰撞惩罚 | **-10** | — |
| 时间惩罚 | -0.05 / step | — |
| 回报量级 | 个位~几十 | **上百(vf_clip 已被迫提到 80)** |

### 问题
奖励尺度大一个数量级 → 价值函数要拟合的目标范围大 → `vf_loss` 难降、优势估计噪声大。
**8 车时优势估计噪声本就因信用分配难而放大,大奖励尺度雪上加霜。**

### 改进
- **将奖励整体缩放到 ±10~20 量级**,对齐参考仓库。例如把各分量权重统一除以一个系数,
  使单 episode 回报落在 [-20, +20]。
- 缩放后 `vf_clip_param` 可回到 10~20,价值函数更易收敛。
- 注意:奖励缩放后要重新跑 Stage1-3 验证(它们目前在大尺度下是好的,缩放是全局改动)。

## 四、可借鉴:网络结构(参考仓库的 4 个设计)

| 设计 | 参考仓库 | 本项目 | 评估 |
|---|---|---|---|
| LIDAR 编码 | **1D CNN** 处理 120 射线(3 通道:障碍/边界/邻居) | GAT 对邻居建图 | 参考的 CNN 对原始激光空间结构利用更充分,可作为补充分支 |
| 中心化 critic | Multi-head attention over agents | GAT critic | 思路一致,本项目不弱 |
| 时序 | 帧堆叠 `history_length=4` | LSTM | LSTM 理论更强但 8 车高频交互下更难训;帧堆叠更鲁棒 |
| 动作分布 | DiagGaussian,局部坐标系(Y 轴指向目标) | DiagGaussian | 参考的"局部坐标系 + Y 指向目标"简化了策略学习,值得借鉴 |

### 改进(按性价比,先低成本后高成本)
1. **(低成本)动作局部坐标系**:把动作定义在"指向目标"的局部系,降低策略学习难度。
2. **(中成本)给激光加 1D CNN 编码分支**,与现有 GAT 特征融合。
3. **(高成本,可选)用帧堆叠替代 LSTM** 做消融对比,看 8 车下哪个更稳。

## 五、训练层面:8 车从未被完整训练过

### 事实
- 第 1 次 Stage4:305k/600k 步被 early-stop **误杀**(已修:min_steps 自适应到 75%)
- 第 2 次 Stage4:380k/600k 步被**外部中断**(OOM 或手动停;dreamer 占 17GB 内存)
- **两次 8 车都没跑满 600k,8 车从没得到完整训练机会**

### 改进
1. **先保证能连续训完**:错开 dreamer 训练时段,或放宽 `RAY_memory_usage_threshold`,
   确保 600k 步不中断。
2. **8 车可能需要远超 600k 步**:4 车用 180 iter(900k 步)收敛,8 车信用分配更难,
   建议给 1M~1.5M 步。
3. 在 8 车跑完整训练前,**不要下"算法不行"的结论**——这是目前最大的未知数。

---

## 六、方法论:单种子不能作为实验结论

- 当前所有 stage 都是 **n=1 单次运行**,无法区分"算法有效"和"运气好"。
- 修好上述问题后,选定配置跑 **3~5 个随机种子**,报告均值 ± 标准差。
- 这是论文级实验的底线,也是判断 8 车到底行不行的唯一可信方式。

## 七、改进优先级(执行顺序)

| 优先级 | 改进项 | 成本 | 预期收益 |
|---|---|---|---|
| 🔴 P0 | 修起终点采样(固定对角配对 / 降 min_agent_sep / 去扰动) | 低 | 任务分布稳定,直接影响可学性 |
| 🔴 P0 | 保证 8 车连续训完(解决 OOM/中断) | 低 | 排除"没训完"这个最大未知数 |
| 🟠 P1 | 奖励缩放到 ±10~20 | 中 | 价值函数易收敛,8 车梯度更稳 |
| 🟠 P1 | 8 车训练步数提到 1M+ | 低 | 给信用分配足够时间 |
| 🟡 P2 | 动作局部坐标系 | 低 | 降低策略学习难度 |
| 🟡 P2 | 激光 1D CNN 分支 | 中 | 增强密集场景感知 |
| 🟢 P3 | 多种子实验(3~5) | 高(算力) | 实验结论可信 |

**建议执行路径**:先做两个 P0(采样 + 训完),用单种子验证 8 车能否到正回报。
若能 → 再上 P1 提升上限,最后多种子定稿。若仍不行 → 再考虑 P2 的网络结构改动。

## 八、逐维度深度对比(为什么和参考仓库效果差这么多)

最根本的一句话:**两者不是同一类问题。** 参考仓库是纯 2D 几何快速仿真,可以无脑堆几百万步;
你的是 Gazebo 物理引擎 + ROS2,step=0.1s 真实仿真,慢约百倍,8 车两次都没跑满 600k 步。
在此基础上,四个维度各有具体差距:

### 8.1 动作空间:差速驱动 + Safety Shield vs 全向速度

| | 参考仓库 | 本项目 |
|---|---|---|
| 动作 | `Box(2)=(vx,vy)` 全向速度 | `Box(2)=(linear_vel, angular_vel)` 差速驱动 |
| 坐标系 | 局部系,Y 轴指向目标 | 机器人体坐标系 |
| 动作→位移 | 线性直接(×max_speed) | 非线性(需先转向再前进) |
| 额外干预 | 无 | **Safety Shield 改写动作** |

**关键差距**:
- 差速驱动(非完整约束,不能横移)本身比全向移动难学得多。
- **Safety Shield 会改写策略输出的动作**(`apply_action` 中 `_apply_safety_shield`)。这破坏了
  PPO 的 on-policy 假设——**策略学到的动作和环境实际执行的动作不一致**,产生 off-policy 偏差。
  8 车时碰撞风险高,shield 触发频繁,偏差被放大,梯度方向被污染。
- 参考仓库动作直接执行,无此问题。

### 8.2 观测空间:维度随车数膨胀 vs 固定射线数

| | 参考仓库 | 本项目 |
|---|---|---|
| 结构 | 60-120 射线×3 通道 + 状态(6) + 堆叠 4 帧 | base 155/agent + 邻居 + localmap(2048) + 全局拼接 |
| 8 车总维度 | **固定**(射线数不随车数变) | base155 + 全局状态(155×8=**1240**) + localmap → **>3000 维** |
| 激光利用 | 1D CNN 提取空间结构 | Top-K 障碍编码(信息有损) |

**关键差距**:
- 你的观测含**全局状态拼接**(`global_state_dim = num_agents × base_obs_dim`),8 车时这一项就 1240 维,
  **观测维度随车数线性膨胀**,大量冗余,信用分配在高维输入下极难。
- 参考仓库射线数固定,无论几车输入维度不变,网络负担恒定。

### 8.3 奖励函数:密集多项 shaping + 惩罚过重 vs 简单稀疏

| | 参考仓库 | 本项目 |
|---|---|---|
| 项数 | 4 项 | 6 项正交,每项多子项 |
| 到达 | +10 | +20 |
| 碰撞 | -10 | **-50(8 车密度自适应 ×1.2 ≈ -58)** |
| 惩罚/奖励比 | 1:1 | **2.5:1(碰撞远大于到达)** |
| 回报量级 | 个位~几十 | 上百 |

**关键差距(这是 8 车负回报的直接原因)**:
- **碰撞惩罚 -50 是到达奖励 +20 的 2.5 倍**。8 车碰撞频繁,智能体的最优策略变成
  **"宁可不动也别撞"**——趋避压倒趋利。这正好解释了 Stage4/6 的全程负回报 + 不到达
  (`r_goal` 下降、`dist_to_goal` 增大)。
- 6 项 shaping 在 8 车时各自的噪声叠加,真实的"到达目标"信号被淹没。
- 奖励量级上百导致价值函数难拟合(见第三节)。

### 8.4 训练方式:受限于仿真速度,8 车从未训完

| | 参考仓库 | 本项目 |
|---|---|---|
| lr | 5e-4 | 3e-4(带衰减) |
| batch | 128 | 5000 |
| curriculum | 无,直接训目标场景 | 6 阶段 |
| 8 车实际步数 | 可堆几百万 | **~380k(没跑满 600k)** |

**关键差距**:
- batch=5000 在 8 车 × 慢仿真下,采样一个 batch 极慢,单位时间更新次数远少于参考仓库。
- 8 车是 curriculum 第 4 阶段,前几阶段策略可能已过拟合低密度场景。
- 最致命:**8 车从没训完**,而参考仓库可以无脑堆步数。这是受 Gazebo 速度所限的结构性劣势。

### 8.5 综合结论

效果差这么多,**不是单一原因,是四个维度的劣势叠加**,且都被"8 车 + 慢仿真 + 没训完"这个
大背景放大。按影响排序:

1. **奖励失衡(碰撞 -50 ≫ 到达 +20)** → 直接导致负回报、不敢动 —— 最该先改。
2. **Safety Shield 破坏 on-policy** → 8 车时梯度被污染 —— 需评估是否纳入训练或改为观测特征。
3. **观测维度随车数膨胀** → 高维信用分配难 —— 中期优化。
4. **没训完 + 任务分布不稳(见第二节)** → 最大未知数 —— 先排除。

> 注意:参考仓库的"简单"是它能跑通 8 车的原因,但也意味着它**没有**你项目的 sim2real 能力
> (真实物理、差速驱动、激光感知、安全保障)。**不要为了跑通 8 车把这些都砍掉**——
> 那会丢失你工作的核心价值。正确方向是:在保留 sim2real 要素的前提下,修复奖励失衡、
> 评估 shield 的训练影响、给 8 车足够训练步数。

## 附:参考仓库关键配置(circle_marl.yaml,8 车对角换位)

```yaml
num_rays: 120
num_agents: 8                  # 圆形对角换位
boundary: 0.1~0.9 (0.8x0.8 归一化)
agent: radius=0.02, max_speed=0.2, fov=360°, max_range=0.3
排布圆半径: 0.25
reward: 到达 +10 / 碰撞 -10 / 时间 -0.05
network: 1D-CNN(激光) + MLP(状态) → 384维 → attention critic + MLP actor
history_length: 4, batch_size: 128, lr: 5e-4
无 curriculum(直接训目标场景)
```
