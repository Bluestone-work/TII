## 🔍 根本原因分析

### 原因1: 熵过高且不下降 ⚠️⚠️⚠️

**数据**：
```
最后 10 次平均熵: 3.058
崩溃时熵: 3.29-3.36

正常收敛值应该: <2.0
```

**问题**：
- 熵 >3.0 意味着策略极度随机
- 策略不确定性太高 → 行为不稳定
- 相当于"完全随机探索"状态

**表现**：
- ✗ 胡乱转向（高熵 → 动作随机）
- ✗ 直接冲撞（没有学到避障策略）

---

### 原因2: VF Loss 失控 ⚠️⚠️

**数据**：
```
VF Loss 统计:
  均值: 4.43
  最大值: 11.45
  崩溃时: 3.25-3.91
  
对比 Policy Loss:
  均值: -0.022
  
VF Loss / Policy Loss 比值: ~200×
```

**问题**：
- VF Loss 远大于 Policy Loss
- Critic 预测不准 → Advantage 估计错误
- Policy 更新方向错误

**因果链**：
```
VF Loss 大
  ↓
Advantage = Q(s,a) - V(s) 估计不准
  ↓
Policy Gradient 有偏
  ↓
策略更新错误
  ↓
性能下降
```

---

### 原因3: Entropy Coefficient 太小 ⚠️⚠️

**当前配置**：
```python
"entropy_coeff": 0.003
"entropy_coeff_schedule": [
    [0, 0.003],
    [150_000, 0.001],
    [250_000, 0.0005],
]
```

**问题**：
- `entropy_coeff = 0.003` 太小
- 策略不在乎熵高低
- 无法驱动熵下降

**PPO Loss**：
```python
total_loss = policy_loss - 0.003 * entropy + vf_loss

# entropy = 3.3 时
entropy_bonus = 0.003 * 3.3 = 0.01  # 太小！

# VF Loss = 3.5 时
total_loss ≈ -0.03 - 0.01 + 3.5 = 3.46  # VF Loss 主导
```

**结果**：
- Policy 更新被 VF Loss 的梯度"带偏"
- Entropy 项贡献可忽略不计
- 策略无法收敛

---

### 原因4: 恶性循环 ⚠️⚠️⚠️

**崩溃的恶性循环**：

```
1. 熵过高（3.3）
   ↓
2. 策略随机 → 收集的轨迹质量差
   ↓
3. Critic 难以学习 → VF Loss 增大
   ↓
4. Advantage 估计错误 → Policy 更新错误
   ↓
5. 策略变差 → 奖励下降
   ↓
6. 奖励下降 → Critic 目标变化 → VF Loss 更大
   ↓
7. 回到步骤 1，循环恶化
```

**证据**：
- Iter 71-83：奖励从 +50 → -5（策略变差）
- 同期熵保持在 3.29-3.36（没有下降）
- 同期 VF Loss 3.25-3.91（保持高位）

#### 方案2: 大幅提高 Entropy Coefficient ⭐⭐⭐⭐⭐

**问题根源**：`entropy_coeff = 0.003` 太小

**修复**：
```python
# train_gnn_mappo_full.py:963-968

# 当前（导致崩溃）
"entropy_coeff": 0.003,
"entropy_coeff_schedule": [
    [0, 0.003],
    [150_000, 0.001],
    [250_000, 0.0005],
],

# 修复（强制降熵）
"entropy_coeff": 0.02,  # 从 0.003 提高到 0.02
"entropy_coeff_schedule": [
    [0, 0.02],          # 强制降熵
    [50_000, 0.01],
    [100_000, 0.005],
    [150_000, 0.002],
    [250_000, 0.001],
],
```

**原理**：
- 更大的 `entropy_coeff` → 策略更在乎熵
- 强制策略降低熵 → 减少随机性
- 策略确定 → 性能稳定

**预期效果**：
- 50k 步内熵从 3.3 降到 2.5
- 100k 步内熵降到 2.0
- 策略稳定，不再崩溃

---

#### 方案3: 降低 VF Loss 权重 ⭐⭐⭐⭐

**问题**：VF Loss 主导训练

**修复**：
```python
# train_gnn_mappo_full.py

# 当前
"vf_loss_coeff": 1.0,  # 默认值

# 修复
"vf_loss_coeff": 0.3,  # 降低到 0.3
```

**原理**：
- VF Loss 影响降低 70%
- Policy Loss 主导训练
- Policy 不被 Critic 的错误带偏

---

#### 方案4: 降低学习率 ⭐⭐⭐

**可能问题**：学习率太高导致策略震荡

**原理**：

- 避免大幅震荡
- 更稳定的收敛

## 💡 为什么会崩溃？

### 类比：学生学习过程

**正常收敛**（Stage2_o, Stage3_o）：
```
阶段1: 随机尝试（熵高）→ 探索
阶段2: 找到规律（熵降）→ 学习
阶段3: 熟练应用（熵低）→ 掌握
```

**新版崩溃**：
```
阶段1: 随机尝试（熵高）→ 探索
   ↓ （entropy_coeff 太小，无法进入阶段2）
停滞: 一直随机（熵不降）→ 困在探索阶段
   ↓ （VF Loss 大，Advantage 错误）
崩溃: 越学越差（负奖励）→ 忘记之前学到的
```

**结论**：
1. `entropy_coeff = 0.003` 对两个版本都太小
2. Stage2_o 靠"运气"（训练够久）最终收敛
3. 新版训练中断 + 不稳定 → 崩溃
