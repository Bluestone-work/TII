## 🔍 关键发现

### ✅ 成功案例的共同特征（Stage2_o, Stage3_o）

1. **熵稳步下降**
   - Stage2_o: -0.0055/iter
   - Stage3_o: -0.0191/iter（更快）
   - 熵趋势始终为负

2. **熵与奖励强负相关**
   - Stage2_o: -0.560
   - Stage3_o: -0.945（非常强）
   - 说明：策略越确定（熵低）→ 性能越好（奖励高）

3. **充分训练**
   - Stage2_o: 900k 步，完整收敛
   - Stage3_o: 690k 步，收敛到更低熵

4. **熵变化轨迹**
   ```
   Stage3_o (最成功):
   Iter   1: 4.02 → 探索
   Iter  61: 2.65 → 过渡
   Iter 101: 1.96 → 收敛
   Iter 121: 1.73 → 稳定
   ```

---

### ❌ 失败案例的特征（Stage2 新版）

1. **熵不降反升**
   - 趋势：+0.0040/iter
   - 从 2.725 升到 2.924

2. **熵与奖励正相关**
   - 相关性：+0.512
   - 异常！应该是负相关

3. **训练严重不充分**
   - 仅 356k 步（Stage2_o 的 40%）
   - 仅 89 iters（中断）

4. **奖励低且不稳定**
   - 最终奖励仅 9.9
   - 对比 Stage2_o 的 133.6

## 🔎 根本原因分析

### 原因1: 训练中断导致未进入收敛阶段

**证据**：
```
Stage2_o (成功):
  0-300k 步: 熵 3.1 → 3.5 (探索阶段，熵略微上升)
  300k-600k: 熵 3.5 → 2.5 (过渡阶段，熵开始下降)
  600k-900k: 熵 2.5 → 2.3 (收敛阶段，熵稳定降低)

Stage2 新版 (失败):
  0-356k 步: 熵 2.7 → 2.9 (停在探索阶段！)
  未进入过渡和收敛阶段
```

**结论**：新版训练在 356k 步中断，还在探索阶段，熵正常应该在此阶段略微上升或持平。

---

### 原因2: Entropy Coefficient Schedule 在早期阶段

**当前 Schedule**：
```python
"entropy_coeff_schedule": [
    [0, 0.003],
    [150_000, 0.001],
    [250_000, 0.0005],
]
```

**新版在 356k 步时的状态**：
- 0-150k: entropy_coeff = 0.003
- 150k-250k: entropy_coeff = 0.003 → 0.001（线性插值）
- 250k-356k: entropy_coeff = 0.001 → 0.0005

**问题**：
- 在 356k 步时，entropy_coeff ≈ 0.0007
- 还没有充分时间让这个小的系数发挥作用
- 熵的下降需要在更小的 entropy_coeff 下训练更长时间

**对比 Stage2_o**：
- 训练了 900k 步
- 其中 650k 步（72%）在 entropy_coeff = 0.0005 下训练
- 有充分时间让熵降低

---

### 原因3: Stage3 的配置可能有所不同

**Stage3_o 的优异表现**：
- 熵下降最快：-0.0191/iter
- 熵-奖励相关性最强：-0.945
- 最终熵最低：1.831

**可能的差异**：
1. entropy_coeff 的初始值更高？
2. entropy_coeff_schedule 更激进？
3. 其他超参数（lr, clip_param）调整？

## 💡 关键洞察

### 1. 熵下降需要时间和小的 entropy_coeff

**Stage2_o 的成功**：
- 900k 步训练
- 其中 650k 步在 entropy_coeff = 0.0005 下
- 有充分时间让策略收敛

**Stage2 新版的问题**：
- 仅 356k 步
- 大部分时间 entropy_coeff 还在 0.001-0.003
- 未进入收敛阶段

### 2. Stage3 更难但熵下降更快

**Stage3_o 的优势**：
- 熵下降速度：0.296/100k步（Stage2_o 的 3.4×）
- 最终熵更低：1.831 vs 2.284
- 熵-奖励相关性更强：-0.945

**可能的原因**：
1. 更难的环境强迫策略更快收敛
2. 使用了更优的 entropy_coeff_schedule
3. 其他超参数调优

### 3. 训练不充分会导致熵异常

**熵与奖励正相关（+0.512）是训练不充分的标志**：
- 正常：策略越确定（熵低）→ 性能越好
- 异常：还在探索阶段，随机尝试偶尔获得高奖励
